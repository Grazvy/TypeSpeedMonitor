Privacy

Application

Security and privacy protection is the most important aspect of developing TypeSpeedMonitor, since creating typing speed statistics requires knowledge about the type of keys pressed, which can be used maliciously to access private information. For this reason, we made this project open source.

The following points show how your privacy preserved:

1. Input processing

A pressed key triggers the event handler which executes the following steps:

  • Check if the key type is a character/symbol or a special key
  • A character/symbol should be included in the WPM statistics, so the time difference to the previous key press is determined
  • Once the current time enters a new 5-second interval, all previous recordings are aggregated and the mean is stored in the database

2. Only aggregated time data is stored

TypeSpeedMonitor does not store any specific keys pressed. It only collects the time difference between keys pressed, aggregates the mean of a 5-second interval and stores the value with the current timestamp in a local sqlite database.

In other words, the application only contains the time difference of all your keystrokes within the current 5-second interval and the database only contains the aggregated mean of each recorded 5-second interval.

3. Data stays local

The application runs local only, no connection to the internet. All the data is stored in a local sqlite database, which can also be accessed in the corresponding application folder.

4. Passwords don't trigger event listener (Mac)

The application runs with user privileges, therefore it cannot receive keystrokes during password inputs or similar actions when Secure Keyboard Entry is enabled.

Additionally: If we see that this application gains more attention, we will invest our money for code-signing to provide additional protection through confirmation of our identities.
